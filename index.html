「Webコンテンツ作成」の成果

本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得することが目的です。 到達目標は次の4つです。(1) 手順書を読み、実験を行い、レポートやプレゼンテーション により報告するという実験の一連の流れを実践できる、（2）コミュニケーション能力の重 要性を理解し、グループワークを円滑に実施できる、（3）モーション解析・編集ができる、 （4）HTML等を用いたWebページ作成ができる。

プロジェクト課題について

プロジェクト課題では特定の地名を入力し検索した際にその場所の緯度、経度、天気、最低 気温、最高気温、湿度、風速、風光、都市名の9つの項目を表示するWebサイトを作りまし た。今回はカイロ、モスクワ、ヨハネスブルグ、北京、東京、シンガポール、シドニー、ロ ンドン、パリ、リオデジャネイロ、ニューヨーク、ロサンゼルスの12個の地名に対応したシ ステムになっています。2回目以降の検索をする際に情報が上書きされるように作ることが 難しかったです。

プロジェクト課題で作成したページ:project.html

数当てゲーム

1~10の数字を3回以内で当てるゲームを作りました。1回ごとに予想値を入力し、予想と答えの大小を表示することでヒントを提示します。１、2回目、3回目、4回目以降でメッセージを変更しています。１、2回目はヒントの表示を3回目は答えを表示、4回目以降は回答とすでにゲームが終わっていることを表示します。回答用の検索場所に入力された値はそのまま数値として扱うことができず、java言語と同じように型変換を行う必要がありました。そのためそのままの型で比較演算を行うことができず、答えに対する代償を判別する際の比較演算子が使えず苦戦しました。また、回数に応じてメッセージを変更する機能を作る際に回数のカウントをする機能も苦戦しました。

数当てゲームのページ:kazuate.html

各種業回の学習事項

第1回 WWW と HTML

今回はGitHub、VisualStudioCodeなどのこの講義で使用されるシステムの使い方を学習しました。また、VisualStudioCodeを使いHTMLの要素としてp要素やbody要素、head要素などの使い方を学びました。VisualStudioCodeでjava言語以外の内容を扱うことが初めてだったのでやり慣れず難しかったです。

演習の成果:演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

今回はCSSによるHTMLの装飾の方法について学習しました。HTML内の情報をWebサイト上で表示する際にCSSに情報を記入しておくことでWebサイト上の文字列や背景などの装飾を行うことができることを学びましたHtmlと入力形式が違いjavaに近い形だったため間違いやすく苦戦しました。

演習の成果:練習問題2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScriptの基本

今回はJavaScriptについて学習しました。JavaScriptはWebページを開く時や、ボタンを押したタイミングでプログラムが実行されページに動きをつけることができるものでした。JavaScriptはCSS以上にjavaに近くjavaとの違いを覚えることに苦戦しました。

演習の成果:練習問題3-6

第4回 DOM と JavaScript

今回はDocument Object Modelを利用したプログラミングについて学習しました。DOMは使用することでWebコンテンツの内容を読み取ったり、コンテンツに変更を加えたりするプログラミングが書けるため、htmlやcssに直接記入せずとも内容を変更することができました。

演習の成果:練習4-5: 箇条書きの追加

第5回 イベント処理

今回はイベント処理について学習しました。イベント処理はマウスやキーボードの操作などを行った際にそれを判断して特定のプログラムを作動させるシステムです。これにより文字入力などの構造を作ることができました。

演習の成果:練習5-3

まとめ

工夫点は主に２つあります。１つ目はページ全体をシンプルな配色や配置にすることでチカチカとうるさい印象にならないように心がけました。２つ目はproject課題の検索窓です。実際に文字を入力してもらう形のページを作ったので英語表記、日本語表記、国名、都市名全てでヒットするように努力したことです。 苦心点は主に３つあります。１つ目はとにかく新たに触れる要素が多くあり、覚えることも多くあったことです。今まで行っていた講義と違い実習だったこともあり、その場その場で覚えていかなければならず、さらにはその場では覚えきれずとても苦労してしまいました。２つ目はプログラミングの講義で学んだJava言語と類似している部分が多くあり、誤ったプログラムを書いてしまうことが多くあったことです。VScodeを使用していることもありプログラムを記入する際にjava言語を入力しそうになることが多くありました。３つ目はプログラムが誤っていた時にどこが誤っているのかを発見することはできましたが、修正することが苦手で時間がかかってしまっていたことです。誤っている箇所はFireFoxが示してくれたのでわかりやすかったですが型変換などパッと改善方法が判断できず、視野が狭くなってしまうことが多くありました。４つ目は、cssのプログラムです。FireFox上でのエラーの発見方法がよく分からず判別に苦戦してしまいました。結果的には何度もコミットしてうまくいくまで繰り返してしまっていたので今後でも判別方法は見つけようと思っています。 強調点は工夫点にも書きましたが見やすい配色を心がけました。 感想です。この講義は私にとってはとても難しく細かい工夫をしている余裕がないほど苦戦していました。教授や先輩、友人に質問し教えてもらうことが多くありましたが教えてもらっても理解をすることは難しく時間がかかってしまい、時間をとらせてしまっていたことに申し訳なさを感じることがありました。そのため、全ての内容に触れた今でも全てを理解できている自信が持てないほどです。空き時間のほとんどをかの講義の学習に費やしたつもりではありましたが課題の完成もギリギリになってしまいました。しかし、反対に長い時間をかけて理解することのできた部分も多くあるので身についた部分が今後の人生の役に立ってくれたら嬉しいと思っています。実習の科目なので苦労もありましたが達成感も多くあり楽しむことも、技術も身につけることもできたと思うので良かったと思っています。
